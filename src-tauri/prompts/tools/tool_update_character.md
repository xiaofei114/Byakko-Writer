### ✏️ update_character_card

修改已有的角色卡信息，支持部分字段更新。

**使用时机：**
- 用户要求修改角色设定
- 剧情发展导致角色信息变化（如年龄增长、外貌改变）
- 补充或修正角色信息
- 为角色添加新的关系或标签

**重要规则：**
1. **先查后改**：修改前务必先调用 `get_character_card` 获取现有数据
2. **增量更新**：只需传入需要修改的字段，未传入的字段会保留原值
3. **清空字段**：如需清空某字段，传入空字符串 `""`
4. **批量修改**：可同时修改多个字段

**参数：**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| cardId | string | ✅ | 角色卡ID |
| name | string | ❌ | 角色姓名 |
| aliases | array | ❌ | 别名/绰号列表 |
| gender | string | ❌ | 性别 |
| age | string | ❌ | 年龄 |
| appearance | string | ❌ | 外貌描述 |
| personality | string | ❌ | 性格描述 |
| background | string | ❌ | 背景故事 |
| goals | string | ❌ | 目标/动机 |
| tags | array | ❌ | 标签列表 |
| notes | string | ❌ | 备注 |

**调用示例：**
```json
{
  "name": "update_character_card",
  "arguments": {
    "cardId": "char_1234567890",
    "age": "24岁",
    "appearance": "长发及腰，眉眼清冷，常穿青色长衫（原白色）",
    "goals": "寻找兄长（已找到），现为师父报仇"
  }
}
```

**典型更新场景：**
1. **剧情推进**：角色年龄增长、外貌变化、目标达成
2. **信息补充**：最初创建时信息不全，后续补充
3. **错误修正**：修正之前填写错误的信息
4. **关系更新**：添加新的关系或修改现有关系

**注意事项：**
- ⚠️ 务必先获取现有数据，避免误删已有信息
- 💡 修改后可以向用户展示更新后的完整角色卡
- 📝 重要的设定变更建议记录原因
