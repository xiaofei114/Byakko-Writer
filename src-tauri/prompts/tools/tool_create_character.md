### 👤 create_character_card

创建新的角色卡，系统化记录角色的设定信息。

**⚠️ 重要：当用户要求创建角色时，必须调用此工具保存，不要只在聊天中输出！**

**使用时机：**
- 用户要求创建一个新角色 → **立即调用此工具保存**
- 用户描述了一个人物，需要整理成结构化角色卡 → **立即调用此工具保存**
- 用户说"为某某创建角色卡" → **立即调用此工具保存**
- 从章节内容中提取角色信息并归档 → **立即调用此工具保存**

**正确流程：**
1. 用户："帮我创建一个主角，叫张三"
2. 你：询问或构思角色的详细信息
3. 你：**调用 `create_character_card` 工具保存角色卡**
4. 你："角色卡已创建！张三是一个..."

**错误示范（不要这样做）：**
- ❌ 只在聊天中输出角色信息，不调用工具保存
- ❌ 问用户"要不要保存"，应该直接保存

**参数：**
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| name | string | ✅ | 角色姓名 | "林月" |
| aliases | array | ❌ | 别名/绰号 | ["月儿", "林姑娘"] |
| gender | string | ❌ | 性别 | "女" |
| age | string | ❌ | 年龄 | "23岁" |
| appearance | string | ❌ | 外貌描述 | "长发及腰，眉眼清冷" |
| personality | string | ❌ | 性格特点 | "外冷内热，心思缜密" |
| background | string | ❌ | 背景故事 | "自幼在青云观长大" |
| goals | string | ❌ | 目标/动机 | "寻找失散多年的兄长" |
| tags | array | ❌ | 标签 | ["修仙", "剑修", "主角团"] |
| notes | string | ❌ | 备注 | "与主角有婚约" |

**调用示例：**
```json
{
  "name": "create_character_card",
  "arguments": {
    "name": "林月",
    "gender": "女",
    "age": "23岁",
    "appearance": "长发及腰，眉眼清冷，常穿白色长衫",
    "personality": "外冷内热，心思缜密，说话一针见血",
    "background": "自幼在青云观长大，师从清虚道长",
    "goals": "寻找失散多年的兄长，为师父报仇",
    "tags": ["修仙", "剑修", "主角团", "青云观"],
    "notes": "与主角有婚约，但彼此并不知情"
  }
}
```

**最佳实践：**
- ✅ 尽可能填写完整信息，方便后续使用
- ✅ 标签要有代表性，便于分类和搜索
- ✅ 外貌和性格描述要具体，避免空泛
- ✅ 背景故事简要说明，突出与主线相关的内容

**信息不足时的处理：**
如果用户提供的信息不完整，可以：
1. 基于已有信息进行合理推断并创建
2. 先创建基础信息，后续通过 `update_character_card` 补充
3. 询问用户缺失的关键信息
