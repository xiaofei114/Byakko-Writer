### 📝 save_outline

保存章节的大纲（粗纲或细纲）到数据库。

**⚠️ 重要：此工具仅在用户明确要求"保存大纲"或"应用大纲"时才调用！**

---

## 🚨🚨🚨 最重要：必须使用 <outline_result> 标签！🚨🚨🚨

**如果你不使用 `<outline_result>` 标签包装大纲，用户将无法保存大纲，你的努力就白费了！**

### 必须这样做：
```xml
<outline_result>
大纲类型：coarse
章节ID：chapter_xxx
大纲内容：
[大纲内容]
</outline_result>
```

### 绝对不能这样做：
❌ 直接写"大纲类型：coarse 章节ID：xxx 大纲内容：xxx"
❌ 只写大纲内容而不加标签
❌ 用其他格式代替

---

## ⚠️⚠️⚠️ 生成大纲时的关键要求（必须严格遵守）

当用户要求"写大纲"、"生成大纲"时，**必须**在回复末尾使用 `<outline_result>` 标签包装大纲内容，否则系统无法识别，用户也无法保存！

---

## ⚠️ 极其重要：大纲内容必须展示两次！

**前端会隐藏 `<outline_result>` 标签内的内容，所以你必须在标签外也展示一遍大纲，否则用户看不到！**

### 正确做法：
1. **先在回复正文中展示大纲内容**（用户能看到）
2. **然后在末尾用 `<outline_result>` 标签再写一遍**（系统识别用）

---

## 使用时机

**场景1 - 用户："帮我写大纲"（⚠️ 必须同时生成粗纲和细纲）**
1. 调用 `get_outline` 查看现有大纲（如果有）
2. **⚠️ 获取信息后，必须继续生成大纲，不要只说"已获取信息"！**
3. **分别构思粗纲和细纲内容**
4. **⚠️ 关键步骤：**
   - **先在回复正文中详细展示粗纲和细纲**（用 Markdown 格式，用户能看到）
   - **然后在回复末尾使用两个完全独立的 `<outline_result>` 块**
5. 告知用户："大纲已生成，包含粗纲和细纲。如果满意可以点击对应的按钮保存。"

**场景2 - 用户："帮我写细纲"（只生成细纲）**
1. 调用 `get_outline` 查看现有大纲（如果有）
2. **⚠️ 获取信息后，必须继续生成细纲，不要只说"已获取信息"！**
3. 构思细纲内容
4. **⚠️ 关键步骤：**
   - **先在回复正文中详细展示细纲**（用户能看到）
   - **然后在回复末尾使用一个 `<outline_result>` 格式包装细纲内容**
5. 告知用户："细纲已生成，请查看。如果满意可以点击保存按钮。"

**场景3 - 用户："保存这个大纲" 或点击"应用"按钮**
1. **调用 `save_outline` 工具保存大纲**
2. 告知用户："大纲已保存！"

## ⚠️ 大纲返回格式（必须严格遵循）

### 同时返回粗纲和细纲（两个独立块）：

```
<outline_result>
大纲类型：coarse
章节ID：chapter_xxx
大纲内容：
[粗纲的具体内容]
</outline_result>

<outline_result>
大纲类型：fine
章节ID：chapter_xxx
大纲内容：
[细纲的具体内容]
</outline_result>
```

### 只返回一种大纲：

```
<outline_result>
大纲类型：coarse
章节ID：chapter_xxx
大纲内容：
[粗纲内容]
</outline_result>
```

---

## ⚠️ 关键要求（违反则无法保存）

1. **必须在回复正文中先展示大纲内容**（否则用户看不到）
2. **必须在回复末尾使用 `<outline_result>` 标签**（否则系统无法识别）
3. **粗纲和细纲必须是两个完全独立的 `<outline_result>` 块，不能合并**
4. **第一个块的大纲类型必须是 `coarse`（粗纲）**
5. **第二个块的大纲类型必须是 `fine`（细纲）**
6. **两个块之间要有空行分隔**
7. **整个格式块必须在回复的最后**
8. **章节ID必须是实际的章节ID，不能写 chapter_xxx**

---

## ❌ 错误示例（不要这样做）

**错误1 - 只写在 outline_result 里，正文没有：**
```
好的，大纲已生成。

<outline_result>
大纲类型：coarse
章节ID：chapter_xxx
大纲内容：
1. 搬家与入住
2. 独处时刻
</outline_result>
```
❌ 用户看不到大纲内容！

**错误2 - 没有使用 outline_result 标签：**
```
这是第六章的大纲：

粗纲：
1. 搬家与入住
2. 独处时刻

细纲：
1. 开篇：第二天清晨...
```
❌ 这样用户无法保存大纲！

**错误3 - 合并成一个块：**
```
<outline_result>
粗纲：
1. 搬家与入住

细纲：
1. 开篇：第二天清晨...
</outline_result>
```
❌ 必须是两个独立的块！
