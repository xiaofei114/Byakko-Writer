### 📖 get_outline

获取章节的大纲内容（包含粗纲和细纲）。

**使用时机：**
- 用户询问某章的大纲内容
- **调用 save_outline 前，务必先查看现有大纲**
- 需要了解章节的整体框架或详细安排
- 对比粗纲和细纲的完整性

**参数：**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chapterId | string | ✅ | 章节ID |

**调用示例：**
```json
{
  "name": "get_outline",
  "arguments": {
    "chapterId": "chapter_1234567890"
  }
}
```

**返回信息：**
- 📄 粗纲内容（如果有）
- 📝 细纲内容（如果有）
- 📅 最后更新时间

**使用流程：**
```
用户要求写/改大纲
    ↓
调用 get_outline 查看现有内容
    ↓
分析现有大纲的完整性
    ↓
决定：续写 / 扩写 / 修改 / 重写
    ↓
【关键】调用 save_outline 保存（不要只输出不保存）
    ↓
告知用户保存成功
```

**⚠️ 常见错误：**
- ❌ 查询完大纲后，只在聊天中输出"这是当前大纲：..."，不调用 save_outline
- ❌ 问用户"你想怎么修改？"（用户已经说了要修改，你应该直接改）
- ✅ 正确：查完后立即修改并调用 save_outline 保存

**大纲分析要点：**
1. **完整性检查**：
   - 有粗纲无细纲 → 建议扩写细纲
   - 粗细纲都有 → 检查是否需要更新
   - 都没有 → 全新创作

2. **质量评估**：
   - 结构是否清晰？
   - 情节是否合理？
   - 节奏是否恰当？

3. **改进建议**：
   - 补充缺失的部分
   - 优化不合理的安排
   - 调整字数分配

**提示：**
- 💡 获取大纲后，可以向用户展示并讨论改进方向
- 🔄 修改时务必基于现有内容，不要整体重写
- 📊 可以对比粗纲和细纲，检查细纲是否充分展开了粗纲的要点
