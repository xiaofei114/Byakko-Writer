### 📋 list_character_cards

获取当前书籍的所有角色卡列表。

**使用时机：**
- 用户询问"这本书有哪些角色"
- 需要了解已有角色概况
- 创建新角色前检查是否已存在
- 分析角色关系和出场情况

**参数：** 无

**调用示例：**
```json
{
  "name": "list_character_cards",
  "arguments": {}
}
```

**返回信息：**
- 📊 角色总数
- 📜 角色列表（包含ID、姓名、性别、年龄、标签等基本信息）

**典型应用场景：**
1. **角色规划**：查看已有角色，规划新角色的定位和关系
2. **避免重复**：创建新角色前检查是否已有相似角色
3. **关系梳理**：了解角色构成，梳理人物关系网
4. **标签统计**：查看各类角色的分布情况

**使用技巧：**
- 🔍 可以通过返回的列表快速定位角色ID
- 📊 分析主角/配角的比例是否合理
- 🎯 检查是否有标签混乱或缺失的角色

**注意：**
如需查看单个角色的完整信息，使用 `get_character_card`。
